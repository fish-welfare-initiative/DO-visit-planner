How It Works

This tool uses a two-tier system: a machine learning model predicts day-level dissolved oxygen risk from weather, and a composite scoring system ranks individual ponds for visit prioritization. Source code and data on GitHub.

Tier 1: Day Risk Level

The daily risk banner is driven by an XGBoost regression model (5 decision trees) trained on 2024-2025 Eluru morning DO measurements. The model predicts dissolved oxygen (mg/L) from weather conditions, where lower predicted DO = higher risk.

Weather features (90.5% of model importance): The model uses 19 weather features from the Open-Meteo Forecast API, computed for Eluru coordinates (16.57°N, 81.03°E). These include 3-day rolling averages of temperature, humidity, cloud cover, and wind speed, plus 3-day sums of precipitation, solar radiation, and sunshine duration, plus instantaneous values at 6-8 AM (typical morning visit time).

How predicted DO maps to risk levels:

Predicted DORisk LevelRecommended Action
≥ 3.6 mg/LLOWStandard schedule, 3-4 farms/staff
3.3 – 3.6 mg/LMODERATENormal schedule, prioritize vulnerable ponds
3.0 – 3.3 mg/LHIGHIncrease to 5-6 farms/staff
< 3.0 mg/LCRITICALMaximum visits, 6+ farms/staff

The OOR (out-of-range) threshold for dissolved oxygen is < 3.0 mg/L. The model's base prediction is 3.50 mg/L, adjusted up or down by weather conditions. Key risk drivers: low sunshine, high humidity, low solar radiation, and high temperature.

Tier 2: Pond Vulnerability Score

Each pond receives a vulnerability score (0–100%) based on its farm characteristics. This is calculated independently of the XGBoost model — it uses the farm data directly rather than routing it through the ML prediction.

Why not use the XGBoost model for pond ranking? The 5-tree model is fundamentally a day-level predictor. Weather drives 90.5% of feature importance, so all ponds get nearly identical predictions on any given day (±0.008 mg/L). Farm features need to be used directly to create meaningful pond-level differentiation.

Vulnerability formula:

ComponentWeightLogic
Stocking density45%Percentile rank among all 162 ponds. Higher density = more fish at risk from low DO.
Feed amount (kg)20%Percentile rank. Overfeeding leads to decomposition, which consumes oxygen.
Feed type5–15%Unknown feed: +15%. Other: +8%. Mash: +5%. DORB (baseline): +0%.
Pond depth0–15%Shallower ponds (< 2.5m) heat faster with less DO buffer. Scales linearly.
Baseline5%Even low-risk ponds get occasional visits.

Current range: 21% (WG-ALS1, SD=1,534) to 66% (WG-PRK1, SD=11,765). This 45-point spread provides meaningful differentiation for visit prioritization.

Priority Score & Rotation

The final priority score that determines visit order combines vulnerability with rotation need:

Priority = (Vulnerability × V-weight + Recency × R-weight) / (V-weight + R-weight)

Recency score (0–1): Based on days since last visit, normalized over a 30-day window. Never visited = 1.0 (maximum priority). Visited yesterday = 0.03. Visited 15 days ago = 0.5. This ensures the team rotates across all farms over time.

Frequency capping: Ponds visited within the last N days (default: 5) are excluded entirely from the plan, even if they have high vulnerability. This prevents the same high-risk farms from being visited every day.

Adjustable weights: The vulnerability and recency weights can be tuned in Settings. Equal weights (default) balance "visit risky ponds" with "visit ponds we haven't seen recently."

Geographic Clustering

Ponds are assigned to staff members using geographic clustering for efficient routes:

1. Seed selection: The highest-priority pond becomes Staff A's seed. Subsequent seeds are chosen to maximize geographic spread while still being high-priority (70% distance + 30% priority weighting).

2. Assignment: Remaining ponds are assigned to the nearest cluster (by centroid) with available capacity.

3. Route ordering: Within each cluster, ponds are sorted by nearest-neighbor routing (greedy TSP) to minimize travel distance. Estimated route distance is shown for each staff member.

All 162 ponds are in the Eluru region (lat 16.57–16.71, lon 81.00–81.27), spanning ~15 villages over roughly 25 km.

Model Details

Algorithm: XGBoost Regressor (reg:squarederror)

Hyperparameters: 5 trees, learning rate 0.10, max depth 3, subsample 0.6, L2 regularization (lambda) 5.0

Training data: 2024-2025 Eluru morning DO measurements (~414 visits, excluding follow-ups)

Validation: 2026 Q1 data — AUC 0.704, Top-5% lift 5.20x, Top-10% lift 3.03x

Top features by importance: Sunshine duration (21%), humidity (17%), solar radiation (15%), temperature (8%), wind speed (8%), precipitation (8%), feed amount (5%), stocking density (4%)

Weather data: Open-Meteo Forecast API with 3 days of past data + 10-day forecast

Pond data: 162 enrolled Eluru (WG) ponds from the ARA program. Currently uses a static snapshot; future versions will integrate with the ARA Laravel backend for live data.
